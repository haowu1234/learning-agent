# Analysis Checklist

在输出之前，逐项检查：

1. **目标**
   - 用户到底要摘要、结论、建议，还是完整报告？
   - 输出粒度是否与用户要求一致？

2. **事实提取**
   - 哪些内容是材料中直接出现的事实？
   - 哪些数据、时间、角色、约束最关键？

3. **判断与推断**
   - 哪些结论可以从材料中直接推出？
   - 哪些只是合理推测，需要显式标注“不确定”？

4. **风险与缺口**
   - 材料是否缺少关键背景、定义或结论？
   - 是否存在相互矛盾、信息过期或范围不清的问题？

5. **交付方式**
   - 用户更需要短摘要、要点列表，还是正式汇报？
   - 是否需要在结尾补“下一步建议 / 待确认项”？
