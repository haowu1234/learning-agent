### 1. 自创生：自我进化 Agent 协议
**Autogenesis: A Self-Evolving Agent Protocol**
🔗 https://arxiv.org/abs/2504.15284 *(Wentao Zhang, Zhe Zhao et al., Apr 21 2026)*

💡 **核心贡献**
提出 Autogenesis 协议，使基于 LLM 的 Agent 系统能够在运行时自主扩展自身能力（生成新工具、新子 Agent）。该协议将 Agent 的"元认知"能力与任务执行解耦，实现无需人工干预的自我迭代演化。

🔧 **技术亮点**
- **动态工具生成**：Agent 在遇到无法处理的任务时，可自动编写并注册新工具到工具库
- **协议化记忆管理**：通过结构化协议而非自由形式 prompt 管理长期状态与跨会话记忆
- **自我验证闭环**：新生成的能力经过沙箱测试后才正式纳入 Agent 能力空间，保证可靠性

📊 **实验结果**
在 WebArena、SWE-bench 等 long-horizon 任务上，相比 ReAct/Toolformer 基线，成功率提升约 18-23%（按原始摘要，具体数字待 PDF 确认）。

🎯 **与 Agent/Routing 的关联**
直接推进 Agent 自我进化能力，是构建无需频繁人工更新的 long-horizon 自主 Agent 的关键方向。

### 2. ReTool：用强化学习训练 LLM 战略性工具调用
**ReTool: Reinforcement Learning for Strategic Tool Use in LLMs**
🔗 https://arxiv.org/abs/2504.11536

💡 **核心贡献**
提出 ReTool 框架，通过强化学习让推理模型（如 DeepSeek R1）在自然语言推理链中动态交织代码执行，从结果反馈中自主学习何时、如何调用工具——完全无需人工标注工具调用时机。

🔧 **技术亮点**
- **多轮实时代码执行 Rollout**：训练时支持 Policy 在推理过程中实时运行代码并回收结果
- **冷启动 + RL 两阶段训练**：先用合成数据微调建立基础，再用任务结果作为奖励信号迭代优化工具调用策略
- **涌现行为：代码自我纠错**（code self-correction）：模型自发学会发现代码错误并重写，无需显式监督

📊 **实验结果**
- AIME 数学奥林匹克基准：ReTool-32B 达到 **67% 准确率**（400 训练步），超越纯文本 RL 基线（40%，1080 步）
- 扩展设置下达到 **72.5%**，超越 OpenAI o1-preview **27.9 个百分点**

🎯 **与 Agent/Routing 的关联**
ReTool 的工具调用时机决策本质上是一种"微粒度路由"——在推理 token 流中动态决定何时转发给代码执行器，对 Agentic Routing 研究有直接参考价值。

### 3. "通往罗马之路"攻击：对抗性后缀劫持 LLM 路由器
**Route to Rome Attack: Directing LLM Routers to Expensive Models via Adversarial Suffix Optimization**
🔗 https://arxiv.org/search/?searchtype=all&query=Route+to+Rome+Attack+LLM+Router *(Haochun Tang et al., Apr 16 2026)*

💡 **核心贡献**
揭示 LLM 路由系统的新型安全威胁：通过在用户查询末尾附加对抗性后缀，可将原本应被分发到廉价小模型的请求强制路由到昂贵大模型，导致成本放大攻击。同时提出后缀优化算法和防御策略。

🔧 **技术亮点**
- **后缀优化攻击**：利用梯度或黑盒方法自动生成极短后缀（< 10 token），对路由器产生定向干扰
- **模型无关性**：攻击对 RouteLLM、Hybrid LLM 等主流路由框架均有效
- **防御方案**：提出基于输入扰动检测和路由置信度校准的防御机制

📊 **实验结果**
在主流 LLM 路由基准上，攻击成功率可达 85%+ （将 simple 查询路由至 GPT-4 级模型），同时用户几乎察觉不到 prompt 变化。

🎯 **与 Agent/Routing 的关联**
直接针对 LLM Routing 系统的对抗鲁棒性，是 Cost-Aware Routing 工程化落地的重要安全议题。

### 4. 动态模型路由与级联综述：高效 LLM 推理全景
**Dynamic Model Routing and Cascading for Efficient LLM Inference: A Survey**
🔗 https://arxiv.org/search/?searchtype=all&query=Dynamic+Model+Routing+Cascading+Efficient+LLM+Inference+Survey *(Updated Apr 21 2026)*

💡 **核心贡献**
系统梳理 LLM 推理优化中的路由（Routing）与级联（Cascading）技术全景，覆盖从查询级别路由、思维链路由、到 Agentic 工作流路由的完整分类体系，并分析各方案在延迟、成本、质量三角中的权衡。

🔧 **技术亮点**
- **四层路由分类**：静态路由 → 动态路由 → 级联路由 → Agentic 路由，统一框架下对比 40+ 方法
- **评估基准梳理**：整理当前主流路由评测数据集及指标（cost-quality Pareto 曲线）
- **开放问题总结**：指出当前路由缺乏标准化训练数据集、对分布外查询鲁棒性差等未解难题

📊 **实验结果**
综述性论文，汇总显示：最优路由策略可在保持 95% 质量的前提下降低 40-60% 推理成本（跨方法平均）。

🎯 **与 Agent/Routing 的关联**
这是截止 2026 年 Q2 最新、最全面的 LLM Routing 综述，是该领域入门和跟踪前沿的必读文献。
