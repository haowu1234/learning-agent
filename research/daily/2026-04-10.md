### 1️⃣

📄 **[先选范式，再解题：推理范式路由作为 LLM Agent 的推理时优化]**
（Select-then-Solve: Paradigm Routing as Inference-Time Optimization for LLM Agents）

🔗 https://arxiv.org/abs/2604.06753

💡 **核心贡献**
本文系统比较了 6 种推理范式（Direct、CoT、ReAct、Plan-Execute、Reflection、ReCode）在 4 个前沿 LLM × 10 个 Benchmark 上的表现（约 18,000 次实验），发现没有任何单一范式能够全局最优。在此基础上，提出用轻量嵌入路由器在推理时为每个任务动态选择最优推理范式。

🔧 **技术亮点**
- 构建大规模对比实验（6范式×4模型×10基准），实证"范式互补性"
- 轻量 Embedding-based Router，无需修改基础模型，即插即用
- 引入"Oracle Gap Recovery"指标，量化路由器对理想上限的恢复程度

📊 **实验结果**
- 路由器将平均准确率从 47.6% 提升至 53.1%，超过最优固定范式 50.3% 达 2.8pp
- 最高恢复 37% Oracle Gap；零样本自路由仅在 GPT-5 上有效（67.1%），弱模型全部失败
- ReAct 在 GAIA 上比 Direct 高 44pp，但 CoT 在 HumanEval 上下降 15pp

🎯 **与 Agent/Routing 的关联**
直接定义了"推理范式路由"这一新任务，是 LLM Routing 领域从"模型路由"拓展到"范式路由"的重要里程碑。

### 2️⃣

📄 **[多轮强化学习训练工具调用 Agent：迭代奖励校准方法]**
（Multi-Turn Reinforcement Learning for Tool-Calling Agents with Iterative Reward Calibration）

🔗 https://arxiv.org/abs/2604.02869

💡 **核心贡献**
针对多轮工具调用 RL 训练中稀疏奖励和跨轮信用分配难题，提出 Iterative Reward Calibration 方法，结合 MT-GRPO + GTPO 混合优势公式，消除奖励误对齐问题，首次在 Tau-Bench 上发布 RL 训练结果。

🔧 **技术亮点**
- 首次将 MT-GRPO（多轮 GRPO）与 GTPO（Token 级策略优化）联合应用于工具调用任务
- 通过经验性判别分析（Discriminative Analysis）系统设计每轮稠密奖励
- 引入 LLM-based 用户模拟器构建真实客服场景训练数据

📊 **实验结果**
- Qwen3.5-4B：63.8% → 66.7%（+2.9pp）；Qwen3-30B-A3B：58.0% → 69.5%（+11.5pp）
- 4B 训练后模型超越 GPT-4.1（49.4%）和 GPT-4o（42.8%），体积小 50 倍
- 30.5B MoE 模型接近 Claude Sonnet 4.5（70.0%）

🎯 **与 Agent/Routing 的关联**
为工具调用 Agent 提供可落地的 RL 训练范式，对构建高效小模型 Agent 具有直接工程价值。

### 3️⃣

📄 **[AgeMem：统一长短期记忆管理的 Agent 记忆框架]**
（Agentic Memory: Learning Unified Long-Term and Short-Term Memory Management for LLM Agents）

🔗 https://arxiv.org/abs/2601.01885

💡 **核心贡献**
提出 AgeMem，将长期记忆（LTM）和短期记忆（STM）的管理统一集成到 Agent 策略本身，通过工具化操作（存储/检索/更新/摘要/丢弃）实现端到端可训练的记忆决策，配合三阶段渐进式强化学习策略。

🔧 **技术亮点**
- 将记忆操作暴露为 Tool-Based Actions，由 LLM Agent 自主决策何时、存什么
- 设计 Step-wise GRPO 处理记忆操作带来的稀疏、不连续奖励
- 三阶段渐进式 RL 训练策略：逐步提升记忆操作复杂度

📊 **实验结果**
在 5 个长视野 Benchmark 上一致超越强基线，同时提升任务性能、记忆质量和上下文效率。

🎯 **与 Agent/Routing 的关联**
通过 RL 让 Agent 自主学习"何时记、何时忘"，为长视野任务中的记忆路由决策提供新思路。

### 4️⃣

📄 **[MAFBench：理解多 Agent LLM 框架的统一基准与实验分析]**
（Understanding Multi-Agent LLM Frameworks: A Unified Benchmark and Experimental Analysis）

🔗 https://arxiv.org/abs/2602.03128

💡 **核心贡献**
提出多 Agent 框架架构分类体系和统一评测套件 MAFBench，在受控条件下横向对比多个主流多 Agent 框架，揭示架构选择对延迟、规划准确率和协调成功率的巨大影响。

🔧 **技术亮点**
- 构建覆盖编排开销、记忆行为、规划、专业化分工、多 Agent 协调的五维评测体系
- 标准化执行流水线，排除任务差异，隔离架构效应
- 提供具体架构设计原则和框架选型指南

📊 **实验结果**
- 仅架构差异即可导致延迟增加超过 100 倍
- 规划准确率可降低最多 30%
- 协调成功率从 >90% 跌至 <30%

🎯 **与 Agent/Routing 的关联**
直接指导多 Agent 系统中框架选型（即"架构路由"），为工程实践提供量化依据。

## 🧠 今日洞察

> **趋势一：Routing 的范畴正在快速扩展。** 从最初的"选哪个模型"，到"用哪种推理范式"（Select-then-Solve），研究者正在将路由决策渗透到 Agent 的每一层——范式选择、工具调用、记忆操作，"动态选择"成为贯穿 Agent 设计的核心哲学。

> **趋势二：RL 正在成为 Agent 能力的底层训练范式。** 无论是记忆管理（AgeMem 的 step-wise GRPO）还是工具调用（MT-GRPO + GTPO），稀疏奖励下的多轮 RL 训练正在替代监督微调，成为驱动 Agent 涌现能力的关键路径。

> **趋势三：工程可行性与实验规模并重。** 4B 参数模型通过 RL 超越 GPT-4o，MAFBench 揭示 100x 延迟差异——研究社区越来越注重将理论贡献落地为可复现、可量化的工程结论，"跑通基准"不再是终点，而是起点。
