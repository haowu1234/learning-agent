## 📄 重新思考模型效率：大模型的多智能体推理
**Rethinking Model Efficiency: Multi-Agent Inference with Large Models**
🔗 https://arxiv.org/abs/2604.04929

💡 **核心贡献**
大模型未必低效——在真实 benchmark 上，大模型以更少的输出 token 便能达到与小模型相当甚至更好的性能。基于此，提出一种多智能体推理框架：保留大模型做短回复，同时在必要时从小模型复用关键推理 token，兼顾效率与能力。

🔧 **技术亮点**
- 系统分析 VLM 各组件的端到端延迟瓶颈，发现输出 token 数量是关键
- 大模型 + 短序列 vs. 小模型 + 长序列：前者在延迟和效果上均有优势
- 多智能体框架：大模型主导推理，从小模型借用"关键推理 token"以弥补差距

📊 **实验结果**
在多个真实世界 benchmark 上，复用小模型推理 token 的大模型，能接近大模型独立推理的性能，同时推理效率显著提升。

🎯 **与 Agent/Routing 的关联**
这是 LLM Routing 的一个新视角：不是"路由到哪个模型"，而是"如何让大小模型协作共享推理过程"。对多 Agent 系统的计算效率设计有直接启发。

## 📄 TriAttention：基于三角函数 KV 压缩的高效长推理
**TriAttention: Efficient Long Reasoning with Trigonometric KV Compression**
🔗 https://arxiv.org/abs/2604.04921

💡 **核心贡献**
发现 RoPE 位置编码会导致现有 KV 缓存压缩方法选错重要 key，提出利用 pre-RoPE 空间中 Q/K 向量的集中特性，通过三角级数估计 key 重要性，显著提升长推理效率和稳定性。

🔧 **技术亮点**
- 在 pre-RoPE 空间中观察到 Q/K 向量围绕固定中心高度集中（Q/K concentration）
- 用三角级数刻画"距离偏好"，以 key 的相对位置为评分依据
- 同时利用 Q/K 范数作为重要性辅助信号
- 无需修改模型权重，即插即用

📊 **实验结果**
在 AIME25（32K token 生成）上：
- 相比 Full Attention，实现 **2.5x 吞吐提升** 或 **10.7x KV 内存压缩**，精度持平
- 同等效率下，现有 SOTA 基线精度仅约为 TriAttention 的一半

🎯 **与 Agent/Routing 的关联**
长 context 管理是 Agent 执行长时任务的核心瓶颈。TriAttention 使单张消费级 GPU 也能运行需要长 context 的 Agent，直接降低 Agent 部署门槛。

## 📄 基于置信度动态的大推理模型早停策略
**Early Stopping for Large Reasoning Models via Confidence Dynamics**
🔗 https://arxiv.org/abs/2604.04930

💡 **核心贡献**
大模型推理时经常"过度思考"——正确的推理轨迹往往在中途就已高置信，错误轨迹才会无止境地拖长。CoDE-Stop 利用中间答案置信度动态来自适应决定何时停止推理，无需任何额外训练。

🔧 **技术亮点**
- 系统研究正确/错误推理轨迹的置信度变化规律
- CoDE-Stop：实时监控中间答案置信度，在合理时机提前终止推理
- 纯推理时干预，不改变模型参数，可插入任意现有模型

📊 **实验结果**
- 相比标准全长推理，token 使用量减少 **25–50%**
- 在多个推理和科学基准上，accuracy-compute tradeoff 优于现有早停方法

🎯 **与 Agent/Routing 的关联**
可视为"推理粒度的 Routing"——根据问题难度动态分配计算资源。对需要大量推理调用的 Agent 任务，累积节省十分可观。

## 📄 MemMachine：面向个性化 AI Agent 的保真记忆系统
**MemMachine: A Ground-Truth-Preserving Memory System for Personalized AI Agent**
🔗 https://arxiv.org/abs/2604.04929

💡 **核心贡献**
提出 MemMachine，一种保真记忆系统，能够在个性化 AI Agent 中忠实保存和检索用户的真实偏好与历史行为，避免幻觉式记忆覆盖问题。

🔧 **技术亮点**
- Ground-Truth-Preserving 机制：写入时严格区分事实记忆与推断记忆
- 分层检索：按优先级召回高置信记忆
- 冲突解决策略：检测并处理新旧记忆间的语义矛盾

📊 **实验结果**
在个性化对话和长期任务 benchmark 上，记忆准确率和用户满意度均优于现有 memory-augmented Agent 方法。

🎯 **与 Agent/Routing 的关联**
Memory 是长时 Agent 的核心组件之一。MemMachine 为构建可靠的个性化 Agent 提供了实用的记忆架构参考。
