### 1. FOCAL：面向高效个人桌面摘要的端侧持续活动过滤日志系统

**原文标题**：FOCAL: Filtered On-device Continuous Activity Logging for Efficient Personal Desktop Summarization

🔗 https://arxiv.org/abs/2604.19541

**作者**：Haoran Yin, Zhiyuan Wen, Jiannong Cao 等（香港理工大学 & 中国移动研究院）

💡 **核心贡献**
桌面交互流在端侧进行隐私友好的任务组织，面临 VLM 高计算开销与跨任务上下文污染两大挑战。FOCAL 提出"过滤-规划-记录"统一多智能体架构，将 VLM 调用频次降低 72.3%，同时大幅提升关键信息召回率。

🔧 **技术亮点**
- 轻量级 Filter Agent 负责噪声抑制，避免无效截图进入推理链路
- 纯文本 Brain Agent 做任务归属识别，大幅减少 VLM 调用
- 任务隔离的 Memory Agent 实现上下文一致的摘要生成，防止跨任务污染

📊 **实验结果**
在 DesktopBench（420 个复杂会话 / 2,572 张截图）上：总 Token 消耗降低 **60.4%**，VLM 调用次数降低 **72.3%**；KIR（关键信息召回）从 0.38 提升至 **0.61**；A→B→A 任务中断场景下 Task Acc 保持 **0.81**，而基线崩溃至 0.03。

🎯 **与 Agent/Routing 的关联**
多 Agent 协作架构通过任务路由分工（Filter→Brain→Record→Memory）实现了计算资源的自适应分配，是 LLM Routing 在端侧 Agent 系统的典型落地。

### 2. Mesh Memory Protocol：多智能体 LLM 系统的语义基础设施

**原文标题**：Mesh Memory Protocol: Semantic Infrastructure for Multi-Agent LLM Systems

🔗 https://arxiv.org/abs/2604.19540

**作者**：Hongwei Xu（独立研究，已在 3 个生产环境部署）

💡 **核心贡献**
现有多 Agent 系统缺乏跨会话认知协作能力。MMP（Mesh Memory Protocol）定义了语义层协议，通过四个可组合原语解决：字段级认知状态共享、可追溯来源声明和会话重启后的记忆持久性三大难题。

🔧 **技术亮点**
- **CAT7**：七字段统一认知记忆块（CMB）结构，规范化 Agent 间信息格式
- **SVAF**：逐字段评估接收方角色锚点，实现精粒度选择性接受而非整体拒绝
- **跨 Agent 谱系追踪**：基于内容哈希的父/祖先关系，防止循环引用"回声"问题
- **Remix 机制**：只存储接收方角色化理解后的 CMB，而非原始 peer 信号

📊 **实验结果**
已在 3 个生产环境运行，支撑多 Agent 作为网络中的独立节点进行跨会话持续协作（23 页详细规范，协议 v0.2.3 已开源）。

🎯 **与 Agent/Routing 的关联**
MMP 直接解决 Multi-Agent Memory 共享问题，提供了长时程任务中 Agent 间状态同步的协议层基础设施。

### 3. Chat2Workflow：用自然语言生成可执行可视化工作流的基准

**原文标题**：Chat2Workflow: A Benchmark for Generating Executable Visual Workflows with Natural Language

🔗 https://arxiv.org/abs/2604.19667

**作者**：Yi Zhong, Shuofei Qiao, Ningyu Zhang 等（**浙江大学 & 腾讯**联合）

💡 **核心贡献**
工业级可视化工作流（如 Dify、Coze 平台）仍需大量人工设计。Chat2Workflow 提供首个面向真实业务场景的基准数据集，并配套鲁棒的 Agentic 框架，通过多轮交互与错误修正将工作流生成准确率提升 5.34%。

🔧 **技术亮点**
- 基准来自真实工业业务工作流，生成结果可直接部署到 Dify/Coze
- Agentic 框架内置循环执行错误修正机制，减少 LLM 生成的语法/逻辑错误
- 实验揭示当前 SOTA 模型在复杂或动态需求下的关键差距

📊 **实验结果**
当前最强模型仍难以生成**稳定、正确、可直接部署**的工作流；Agentic 框架带来最高 **5.34% resolve rate** 提升，但真实场景差距依然显著，确立了明确的研究方向。

🎯 **与 Agent/Routing 的关联**
工作流 Agent 的规划与工具调用能力直接对应 Long-Horizon Agent Planning，腾讯参与表明工业界对该方向的高度关注。

### 4. 多 LLM 路由与级联动态模型选择综述（更新版）

**原文标题**：Dynamic Model Routing and Cascading for Efficient LLM Inference: A Survey

🔗 https://arxiv.org/abs/2603.04445

**作者**：Yasmin Moslem 等（Trinity College Dublin & Huawei Ireland，2026-04-21 更新 v2）

💡 **核心贡献**
系统梳理多 LLM 路由与级联推理的前沿方法，提出三维概念框架（何时决策 × 使用何种信息 × 如何计算），覆盖 Query Difficulty / Human Preference / Uncertainty / RL 等路由范式，揭示实际系统的"复合范式"本质。

🔧 **技术亮点**
- 区分"模型内部路由"（MoE）与"跨独立 LLM 路由"，聚焦后者
- 三维框架：when（预推理/推理中/推理后） × what（查询特征/输出信号） × how（规则/学习/级联）
- 分析 Pareto 前沿：在固定成本下比较效能 vs. 在相当效能下比较成本

📊 **实验结果**
综述证明：精心设计的路由系统可以超越最强单体模型的性能，通过跨模型专业化能力的策略性协同实现效能与效率的双重最大化。

🎯 **与 Agent/Routing 的关联**
这是截至2026年4月最全面的 LLM Routing 综述，直接对应研究方向核心，兼顾理论分类与实践部署约束。
