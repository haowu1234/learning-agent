### 📄 **【SoK】Agentic RAG 系统化知识梳理：分类法、架构、评估与研究方向**
（SoK: Agentic Retrieval-Augmented Generation (RAG): Taxonomy, Architectures, Evaluation, and Research Directions）

🔗 https://arxiv.org/abs/2603.07379

💡 **核心贡献**
本文首次对 Agentic RAG 进行系统化知识梳理（SoK），将自主检索-生成循环形式化为有限视野部分可观测马尔可夫决策过程（POMDP），构建了统一的分析框架。研究识别了自主循环中的严重系统性风险，包括幻觉级联传播、记忆污染、检索错位和工具执行漏洞。

🔧 **技术亮点**
- 将 Agentic RAG 的控制策略与状态转移形式化为 POMDP，提供数学严格的系统描述
- 构建覆盖规划机制、检索编排、记忆范式和工具调用行为的综合分类体系
- 识别并分类 Agentic 循环中的系统性可靠性风险（幻觉传播、记忆毒化等）

📊 **实验结果**
综述性论文，无实验数据；提出 cost-aware orchestration、formal trajectory evaluation、stable adaptive retrieval 等研究方向。

🎯 **与 Agent/Routing 的关联**
直接为 Agent 系统中的记忆管理、规划策略与检索路由提供统一的理论框架和开放研究路线图。

### 📄 **ODAR：基于主动推断的 LLM 推理自适应路由**
（ODAR: Principled Adaptive Routing for LLM Reasoning via Active Inference）

🔗 https://arxiv.org/abs/2602.23681

💡 **核心贡献**
ODAR 提出了一种基于「主动推断」（Active Inference）原理的 LLM 路由框架，通过难度估计器动态将查询路由至快速（Fast Agent）或慢速（Slow Agent），以最小化自由能目标函数平衡准确率与效率。本文挑战了"均匀暴力采样（best-of-N）就是最优test-time scaling"的主流假设。

🔧 **技术亮点**
- 基于摊销主动推断（amortized active inference）的难度估计器，实现动态 Fast/Slow Agent 路由
- 自由能原理驱动的风险敏感融合机制：同时最小化对数似然损失和认知不确定性（varentropy）
- 完全支持开源模型（Llama 4 + DeepSeek），路由决策无需商业黑盒 API

📊 **实验结果**
- 跨 23 个基准测试：MATH 达 **98.2%** 准确率，Humanity's Last Exam (HLE) 达 **54.8%**
- 在计算量匹配设置下，与同质采样策略相比，计算成本降低 **82%**
- 路由决策本身仅需 **22.5μs** 延迟

🎯 **与 Agent/Routing 的关联**
直接解决 LLM Routing 核心问题：如何基于问题难度实时分配计算资源，是 Routing 领域中理论最扎实的新方法之一。

### 📄 **ParetoBandit：面向非平稳 LLM 服务的预算感知自适应路由**
（ParetoBandit: Budget-Paced Adaptive Routing for Non-Stationary LLM Serving）

🔗 https://arxiv.org/abs/2604.00136

💡 **核心贡献**
ParetoBandit 是首个能同时满足三大生产约束的开源 LLM 路由器：动态执行美元计价的预算上限、在线适应模型价格/质量的非平稳变化、以及运行时零停机接入新模型。核心思路是将成本感知的 Contextual Bandit 与在线对偶预算控制器结合。

🔧 **技术亮点**
- 在线原始-对偶（primal-dual）预算调节器：闭环控制每请求成本上限，无需离线惩罚调参
- 几何遗忘（Geometric Forgetting）充分统计量：快速适应价格/质量漂移，同时保留离线先验
- 热插拔模型注册表（Hot-swap registry）+ UCB 强制探索：支持运行时添加/删除模型

📊 **实验结果**
- 1,824 条 prompt，三模型组合（成本跨度约 ~530x），跨 7 个预算上限测试
- 平均每请求成本超标幅度 **≤0.4%**（严格预算合规）
- 新冷启动模型约 **~142 步**内达到有效采用率，不违反成本上限
- 端到端路由延迟 **9.8ms**（CPU），路由决策本身仅 **22.5μs**

🎯 **与 Agent/Routing 的关联**
直接解决生产级 LLM 路由的核心痛点：价格波动、模型质量静默回退、新模型上线无感切换，对 Agent 基础设施建设有极高参考价值。

### 📄 **Aragog：面向 Agentic 工作流可扩展服务的即时模型路由**
（Aragog: Just-in-Time Model Routing for Scalable Serving of Agentic Workflows）

🔗 https://arxiv.org/abs/2511.20975

💡 **核心贡献**
Aragog 针对 Agentic 工作流中"请求执行期间系统负载剧烈波动导致固定配置迅速次优"的问题，提出渐进式自适应路由方案：将配置决策分解为一次性精度保持配置识别 + 廉价的逐阶段实时调度。

🔧 **技术亮点**
- 将 Agentic 工作流配置选择解耦为两层：一次性路由（识别精度保持配置集合）+ 逐阶段调度（利用实时系统观测动态选择）
- 逐阶段调度器在大规模配置空间中实现低开销实时决策
- 显式建模 Agentic 工作流的异构性与长期依赖，克服执行前绑定的根本缺陷

📊 **实验结果**
- 跨多种工作流和模型系列测试
- 最大服务吞吐量提升 **50.0%–217.0%**
- 峰值请求率下中位延迟降低 **32.5%–78.9%**
- 准确率与最昂贵配置相当

🎯 **与 Agent/Routing 的关联**
将 Routing 的时间维度从"请求前"延伸到"请求执行全程"，是 Agentic 场景下最具工程落地价值的路由创新之一。

## 🧠 今日洞察

**洞察一：Routing 正从"静态分类"走向"动态自适应"。** 本日三篇 Routing 论文（ODAR、ParetoBandit、Aragog）都指向同一趋势：单次查询前做路由决策已不够用，真正的挑战是如何在模型生态非平稳（价格变动、质量漂移、新模型上线）和 Agentic 工作流长期执行中持续动态调整——这要求路由系统具备在线学习和状态感知能力。

**洞察二：Agent 系统可靠性成为新重心。** Agentic RAG 的 SoK 综述揭示，随着 Agent 的自主性增强，幻觉级联、记忆毒化等系统性风险正成为阻碍规模化落地的核心障碍，学术界开始从"提升能力"转向"构建可控、可验证的 Agent 基础设施"。

**洞察三：Test-time compute scaling 需要路由而非暴力采样。** ODAR 以 82% 的计算成本节约和跨 23 个基准的强劲表现，有力论证了"自适应路由"在 test-time scaling 中的优越性——单纯增加 best-of-N 采样并不是最优路径，基于难度感知的动态资源分配才是。
