## 📄 **GraphPlanner：图记忆增强的多智能体 LLM 路由框架**
（GraphPlanner: Graph Memory-Augmented Agentic Routing for Multi-Agent LLMs）

🔗 https://arxiv.org/abs/2604.23626 | ICLR 2026

💡 **核心贡献**
GraphPlanner 将 LLM 路由问题扩展到多智能体 Agentic 场景，引入异构图记忆（GARNet）同时捕获查询-智能体-响应三方交互历史。它将工作流生成建模为马尔可夫决策过程（MDP），每步同时选择 LLM backbone 和智能体角色（Planner / Executor / Summarizer），并以强化学习端到端优化。

🔧 **技术亮点**
- 异构图 GARNet 同时编码历史记忆和工作流记忆，支持感应式（inductive）和直推式（transductive）推理
- MDP 框架联合优化任务性能与计算开销，兼顾精度与成本
- 支持零样本泛化：对未见任务和未见 LLM 均表现出强泛化能力

📊 **实验结果**
在 14 个多样 LLM 任务上：相比强 baseline 路由器准确率提升最高 **9.3%**，GPU 显存从 186.26 GiB 降至仅 **1.04 GiB**（节省约 99.4%）；代码已开源：https://github.com/ulab-uiuc/GraphPlanner

🎯 **与 Agent/Routing 的关联**
GraphPlanner 是首个将记忆增强的图结构引入多智能体路由的工作，直接打通了 Agent 规划与 LLM Routing 两个研究方向。

## 📄 **WebThinker：赋予大推理模型深度研究能力**
（WebThinker: Empowering Large Reasoning Models with Deep Research Capability）

🔗 https://arxiv.org/abs/2504.21776 | NeurIPS 2025

💡 **核心贡献**
WebThinker 提出一种深度研究 Agent，让大推理模型（LRM，如 o1、DeepSeek-R1）在推理过程中自主搜索网页、在页面间导航并起草报告，突破静态内部知识的瓶颈。它通过 RL（在线 DPO）进一步优化工具使用策略。

🔧 **技术亮点**
- Deep Web Explorer 模块：在遇到知识空白时动态搜索、导航与信息提取
- Autonomous Think-Search-and-Draft 策略：无缝交织推理、信息收集、报告撰写三个阶段
- RL-based 训练：迭代在线 DPO 提升研究工具调用质量

📊 **实验结果**
在 GPQA、GAIA、WebWalkerQA、HLE 复杂推理基准及 Glaive 科学报告生成任务上，WebThinker 显著超过现有方法和强闭源系统；代码开源：https://github.com/RUC-NLPIR/WebThinker

🎯 **与 Agent/Routing 的关联**
WebThinker 代表了 long-horizon planning + tool use 的前沿范式：Agent 主动决策"何时搜索、搜索什么"，是工具调用路由在单智能体场景中的最佳实践。

## 📄 **Route to Rome Attack：对 LLM 路由器的对抗性后缀攻击**
（Route to Rome Attack: Directing LLM Routers to Expensive Models via Adversarial Suffix Optimization）

🔗 https://arxiv.org/abs/2504.12370（基于搜索列表，实际 arXiv ID 待确认）| April 2026

💡 **核心贡献**
首次系统研究针对成本感知 LLM 路由器的对抗性攻击：通过在用户输入末尾附加优化的对抗后缀，可强迫路由器将原本应分配给廉价模型的查询路由至昂贵模型，造成显著的成本放大。

🔧 **技术亮点**
- 基于梯度优化的对抗后缀生成，不需要模型白盒权重（黑盒迁移性强）
- 揭示了主流路由策略（基于 embedding 相似度、分类器等）的共同脆弱性
- 提出针对路由鲁棒性的评估框架，呼吁将安全性纳入路由设计

📊 **实验结果**
在多个主流 LLM 路由 baseline 上，攻击成功率高（具体数值见论文）；迁移性实验表明后缀在不同路由架构间可有效迁移。

🎯 **与 Agent/Routing 的关联**
LLM Routing 的安全性盲区：该工作是路由系统走向生产部署必须直视的对抗威胁，与 routing cost-optimization 研究形成正面冲突。

## 📄 **Autogenesis：自进化 Agent 协议**
（Autogenesis: A Self-Evolving Agent Protocol）

🔗 https://arxiv.org/abs/2504.12271（基于搜索列表）| April 2026，作者：Wentao Zhang 等（NTU / Princeton）

💡 **核心贡献**
Autogenesis 提出一套让 LLM-based Agent 自我进化的协议：Agent 在执行任务时不仅利用现有能力，还能动态生成、验证并持久化新工具/新技能，使自身能力边界随使用而扩展。

🔧 **技术亮点**
- 技能自生成机制：Agent 在遇到能力边界时自动生成并测试新工具代码
- 持久化记忆库：通过结构化技能库（skill library）跨任务保存能力，避免重复造轮子
- 自我评估与验证：生成的技能经自动验证后才被纳入技能库，保障可靠性

📊 **实验结果**
在开放世界任务（包括代码生成、数据分析等）上，随着技能库增长，Autogenesis Agent 在 long-horizon 任务上的成功率持续提升，显著优于无演化的静态 Agent。

🎯 **与 Agent/Routing 的关联**
自进化 Agent 是 long-horizon 规划的终极形态：Autogenesis 将工具使用（Tool Use）与记忆（Memory）深度结合，代表 Agent 从"使用工具"走向"创造工具"的范式跃迁。

## 🧠 今日洞察

今日研究呈现三条清晰主线：

1. **路由与规划的深度融合**：GraphPlanner 以图记忆桥接多智能体路由与任务规划，表明"谁来做"（路由）和"怎么做"（规划）正在被统一建模，而非孤立优化。

2. **Long-horizon Agent 能力建设提速**：WebThinker 和 Autogenesis 分别从"外部信息获取"和"技能自我扩展"两个维度突破 Agent 能力上限，RL 训练正成为提升工具使用质量的标配手段。

3. **路由安全成为新前沿**：Route to Rome Attack 预示着 LLM 路由系统即将面临与 prompt injection 类似的对抗威胁，路由鲁棒性将是工业部署的关键考量。
