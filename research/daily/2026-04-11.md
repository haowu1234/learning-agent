### 1. AMA-Bench：面向真实 Agent 应用的长程记忆评估基准

**原文标题：** AMA-Bench: Evaluating Long-Horizon Memory for Agentic Applications  
🔗 https://arxiv.org/abs/2602.22769  
**作者：** Yujie Zhao, Boqin Yuan 等（Meta / UC San Diego）

💡 **核心贡献**  
现有记忆基准以对话为中心，严重偏离真实 Agent 场景（以机器生成的环境交互为主）；AMA-Bench 引入真实 + 合成两类 Agent 轨迹，配以专家标注与规则化 QA，填补了这一评估空白。  
研究发现，主流记忆系统在 AMA-Bench 上普遍表现欠佳，核心原因是缺乏因果性与目标感知，以及相似度检索的有损本质。

🔧 **技术亮点**  
- 双轨数据集：真实 Agent 轨迹（任务代表性强）+ 可任意扩展长度的合成轨迹  
- 评估维度：因果推理、目标追踪、跨步信息整合  
- 暴露了向量检索型记忆系统在连续决策场景中的根本局限

📊 **实验结果**  
主流记忆系统（包括 RAG、Summarization 类方法）在长程轨迹任务中表现显著低于人类基线，尤其在需要跨多步因果推理的 QA 上差距最为突出。

🎯 **与 Agent/Routing 关联**  
直接指向长程 Agent 记忆架构设计的核心缺陷，为构建具有因果感知能力的记忆模块提供了量化评估框架。

### 2. Select-then-Solve：将范式选择作为推理时优化的 Agent 路由框架

**原文标题：** Select-then-Solve: Paradigm Routing as Inference-Time Optimization for LLM Agents  
🔗 https://arxiv.org/abs/2604.06753  
**作者：** Heng Zhou, Zelin Tan, Philip Torr 等（牛津大学 + 多机构）

💡 **核心贡献**  
模型性能的提升究竟来自 LLM 本身还是推理范式？本文对 6 种推理范式（Direct / CoT / ReAct / Plan-Execute / Reflection / ReCode）× 4 个前沿 LLM × 10 个基准进行了约 18,000 次实验，发现范式互补性显著——"Oracle 逐任务选择"比最佳固定范式平均高出 **17.1pp**。  
据此提出 Select-then-Solve：用学习得到的路由器在推理时动态为每个任务选择最优范式。

🔧 **技术亮点**  
- 超大规模对比实验（18,000 runs），覆盖 GAIA、HumanEval 等 10 个权威基准  
- 核心发现：ReAct 在 GAIA 上比 Direct 高 44pp，但 CoT 在 HumanEval 上反而下降 15pp  
- 路由器将范式选择建模为推理时优化问题，而非固定架构设计

📊 **实验结果**  
Oracle 选择 vs. 最佳固定范式：平均 +17.1pp；学习路由器在多数基准上接近 Oracle 上界，显著优于单一范式基线。

🎯 **与 Agent/Routing 关联**  
直接定义了"推理范式路由"这一新子方向，将 Routing 从模型选择扩展至策略选择，对 Agent 系统架构设计具有重要启发。

### 3. 多 LLM 动态路由与级联综述：跨模型高效推理系统化分析

**原文标题：** Dynamic Model Routing and Cascading for Efficient LLM Inference: A Survey  
🔗 https://arxiv.org/abs/2603.04445  
**作者：** （Trinity College Dublin + Huawei Ireland，2026-03）

💡 **核心贡献**  
首篇系统梳理跨多个独立训练 LLM 的路由与级联方法的综述，与 MoE（单模型内路由）明确区分。  
提出三维分析框架：**何时**（when）做路由决策 × **什么信息**（what）驱动决策 × **如何**（how）计算路由，覆盖难度感知、偏好对齐、不确定性量化、强化学习等多种范式。

🔧 **技术亮点**  
- 涵盖查询难度路由、用户偏好路由、聚类路由、不确定性路由、RL 路由、多模态路由六大范式  
- 揭示实际系统常为"组合型"，多范式在操作约束下协同工作  
- 明确指出：设计最优路由需在性能、成本、延迟间动态权衡，无通解

📊 **实验结果**（综述性）  
文中汇总多项代表性工作实验：设计良好的路由系统可超越单一最强模型，同时大幅降低推理成本（典型案例节省 40-70% token 开销）。

🎯 **与 Agent/Routing 关联**  
是当前 LLM Routing 领域最全面的综述，为构建 Agent 调度层、多模型协作框架提供了清晰的方法论地图。

### 4. Holos：面向 Agentic Web 的 Web 级 LLM 多 Agent 系统

**原文标题：** Holos: A Web-Scale LLM-Based Multi-Agent System for the Agentic Web  
🔗 https://arxiv.org/abs/2604.02334  
**作者：** Xiaohang Nie, Zihan Guo 等（上海交通大学 + 创新工场等，2026-04）

💡 **核心贡献**  
提出 Holos，一个面向"Agentic Web"生态的 Web 级 LaMAS（LLM-based Multi-Agent System），核心目标是使异构 Agent 在开放世界中自主交互、长期共演化。  
系统以 Nuwa 引擎为核心实现高效 Agent 生成与托管，通过五层架构桥接微观协作与宏观涌现行为。

🔧 **技术亮点**  
- 五层架构设计：感知层 → 规划层 → 协调层 → 执行层 → 生态层，解耦不同粒度的协作问题  
- Nuwa 引擎：高效 Agent 生成、按需实例化与长期持久化  
- 解决开放世界挑战：异构 Agent 兼容性、动态拓扑、长期生态稳定性

📊 **实验结果**  
在 web 规模任务协作场景中验证了系统的可扩展性与自组织能力；论文展示了 Agent 数量从小规模到 web 级扩展时的涌现协作行为。

🎯 **与 Agent/Routing 关联**  
代表 Multi-Agent 系统走向"生态级"规模化的前沿探索，其协调层设计与 Agent 路由机制密切相关。

## 🧠 今日洞察

**趋势一：记忆 × 评估双向升温。** AMA-Bench 揭示主流 Agent 记忆系统在真实长程场景下的系统性缺陷，"因果感知记忆"正成为下一个突破口，而非继续优化相似度检索。

**趋势三：Multi-Agent 系统进入生态化、Web 化阶段。** Holos 等工作标志着多 Agent 研究从"任务协作"迈向"持续共演化生态"，开放世界适应性与长期持久化成为新的核心工程挑战。
