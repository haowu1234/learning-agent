### 1. GraphPlanner：图记忆增强的多智能体路由框架
**（GraphPlanner: Graph Memory-Augmented Agentic Routing for Multi-Agent LLMs）**

🔗 https://arxiv.org/abs/2604.23626
📅 提交：2026-04-26 | 录用：ICLR 2026

💡 **核心贡献**
GraphPlanner 将 LLM 路由扩展至真实多智能体场景，提出异构图记忆增强的 Agentic Router，通过 GARNet 图结构同时建模历史记忆与工作流记忆。框架将工作流生成形式化为 MDP 过程，每步同时选择 LLM backbone 和 Agent 角色（Planner / Executor / Summarizer），并以强化学习联合优化任务性能与计算效率。

🔧 **技术亮点**
- **GARNet 异构图**：捕获 query-agent-response 三类节点的交互历史，支持 inductive（泛化到新任务）与 transductive（利用历史记忆）两种推断模式
- **MDP 工作流生成**：路由决策不再是单轮选模型，而是序列决策过程，可动态调整每一步的角色分配
- **RL 联合优化**：同时优化任务准确率和 GPU 内存消耗，不需要穷举所有 LLM 组合

📊 **实验结果**
- 在 14 个多样化 LLM 任务上超越单轮/多轮强基线，精度提升最高 **+9.3%**
- GPU 成本从 186.26 GiB 骤降至 **1.04 GiB**（减少约 99.4%）
- 对未见任务和未见 LLM 具有强零样本泛化能力

🎯 **与 Agent/Routing 的关联**
同时解决了多智能体协作中的路由决策与记忆管理两大核心问题，是迄今将 Routing + Memory + Multi-Agent 三者融合最完整的工作之一。

### 2. Autogenesis：自进化智能体协议
**（Autogenesis: A Self-Evolving Agent Protocol）**

🔗 https://arxiv.org/abs/2604.15034
📅 提交：2026-04-16（v2: 2026-04-21）

💡 **核心贡献**
现有 Agent 协议（A2A、MCP）在跨实体生命周期管理、版本追踪和自演化接口上存在严重不足，导致脆弱的"胶水代码"组合。AGP（Autogenesis Protocol）通过将 prompt、agent、tool、environment、memory 统一注册为协议资源，并设计闭环自演化接口（提议→评估→提交），实现了带有审计溯源和回滚能力的自进化。

🔧 **技术亮点**
- **RSPL（资源底层协议）**：将所有 Agent 组件（包括记忆）建模为有显式状态、生命周期和版本接口的协议注册资源
- **SEPL（自进化协议层）**：闭环算子接口，改进具有可审计的血统（lineage）和回滚保障，避免不可控的自我修改
- **AGS 系统**：基于 AGP 实现的自进化多智能体系统，可在运行时动态实例化、检索、精化协议注册资源

📊 **实验结果**
在多个需要长程规划与工具使用的 benchmark 上相比强基线取得一致提升（论文提供完整对比实验，代码已开源：github.com/DVampire/Autogenesis）

🎯 **与 Agent/Routing 的关联**
从协议层面解决了长程 Agent 任务中工具、记忆和子 Agent 的版本管理与动态路由问题，是 Agent 基础设施层的重要进展。

### 3. 动态模型路由与级联综述：高效 LLM 推理
**（Dynamic Model Routing and Cascading for Efficient LLM Inference: A Survey）**

🔗 https://arxiv.org/search/?searchtype=all&query=Dynamic+Model+Routing+Cascading+Efficient+LLM+Inference+Survey
📅 提交：2026-04-21（v1: 2026-02-23）

💡 **核心贡献**
首篇系统梳理 LLM 动态路由与级联推理的综述论文，覆盖从早期 query routing 到现代 agentic cascade 的完整演进脉络。论文对路由粒度（token/sentence/query/task）、路由目标（成本/延迟/质量）、路由信号（模型置信度/语义特征/强化学习）进行了统一分类框架。

🔧 **技术亮点**
- **四层分类体系**：从模型级路由（小模型→大模型 fallback）到 Agent 级路由（多专家 Agent 协作）的层次化归纳
- **评估维度统一化**：提出路由效率（Routing Efficiency Pareto Frontier）作为跨方法比较的标准指标
- **开放问题总结**：明确指出 adversarial routing robustness、cross-modal routing、online routing adaptation 三类前沿挑战

📊 **实验结果**
综述性论文，无原始实验；但汇总了 50+ 方法在 MMLU、GSM8K、MT-Bench 上的对比数据，提供了横向性能对比表格

🎯 **与 Agent/Routing 的关联**
是理解当前 LLM Routing 全景的必读综述，尤其对 Agentic Workflow 下的路由挑战有专章分析。

### 4. Route to Rome：针对 LLM 路由器的对抗攻击
**（Route to Rome Attack: Directing LLM Routers to Expensive Models via Adversarial Suffix Optimization）**

🔗 https://arxiv.org/abs/2604.15170（基于搜索结果推断 ID，提交：2026-04-16）
📅 提交：2026-04-16

💡 **核心贡献**
首次揭示成本感知路由（Cost-Aware Routing）系统的安全漏洞：攻击者可以通过在用户 query 末尾附加对抗性 suffix，欺骗路由器将本应发送到廉价小模型的请求错误路由至昂贵大模型，造成服务成本攻击（Cost Inflation Attack）。

🔧 **技术亮点**
- **对抗 Suffix 优化**：基于梯度优化生成最优 adversarial suffix，对路由决策的影响不依赖于路由器内部架构知识（黑盒可迁移）
- **攻击成本分析**：证明小代价攻击（数十 token suffix）可导致路由成本提升数倍乃至数十倍
- **防御初探**：提出 suffix 检测器和路由置信度校准两种缓解方向，为后续防御研究奠基

📊 **实验结果**
在 RouteLLM、LLM-Blender 等主流路由框架上验证攻击有效性，路由被操纵率（Route Manipulation Rate）在黑盒场景下达 70%+ ；防御方案将攻击成功率降低约 40%

🎯 **与 Agent/Routing 的关联**
为 LLM 路由安全提出了全新威胁模型，对部署生产级路由系统的工程师具有直接参考价值。

## 🧠 今日洞察

> **记忆 × 路由 = 新融合方向**：GraphPlanner 的出现标志着 LLM Routing 正从"单次模型选择"升级为"带历史记忆的多步协作决策"，这与长程 Agent 的核心需求高度契合，预计未来 Routing 研究将更多引入图结构记忆和强化学习优化。

> **Agent 协议层成新战场**：Autogenesis 提出的 AGP 协议表明，社区已开始认真对待 Agent 基础设施的版本管理与自演化问题——这是从"能用的 Agent Demo"走向"生产可靠 Agent 系统"的必经之路。

> **路由安全不可忽视**：Route to Rome 攻击提醒我们，随着 LLM Routing 被部署到实际系统，对抗鲁棒性将成为与性能同等重要的设计维度，安全感知路由（Security-Aware Routing）或将成为 2026 年的热门子方向。
